# Update Session Documentation

Update SESSION_RESUME.md and LEARNING_PLAN.md with current progress.

## When to Use
- At the end of each session
- After completing a major milestone
- Before switching to a new phase
- When significant learnings are discovered

## Tasks

1. **Review current conversation** for:
   - Tasks completed
   - Decisions made
   - Learnings discovered
   - Challenges overcome
   - Files created/modified
   - Current blockers

2. **Update SESSION_RESUME.md:**
   - Update "Last Updated" timestamp
   - Update "Current State" section
   - Add completed tasks to appropriate sections
   - Update "Next Session Plan" with remaining work
   - Add new "Key Learnings" if any
   - Update "Open Questions" if new ones arose
   - Increment session number if ending session

3. **Update LEARNING_PLAN.md:**
   - Mark completed tasks/objectives
   - Update phase status (pending → in_progress → completed)
   - Add new insights to "Key Insights (Running Log)"
   - Document challenges in appropriate phase sections
   - Update "Learning Metrics" if significant progress
   - Add to "Resources & References" if new materials used
   - Update "Plan Updates" version log

4. **Summarize changes** made to user:
   ```
   # Documentation Updated

   ## SESSION_RESUME.md
   - [List key updates]

   ## LEARNING_PLAN.md
   - [List key updates]

   ## Next Steps
   - [What to do next session]
   ```

5. **Suggest** if any other documentation needs updating:
   - ARCHITECTURE.md (if architecture changed)
   - README.md (if setup changed)
   - New files needed (if patterns emerged)

## Important
- Be concise but comprehensive
- Focus on what's useful for resuming next session
- Capture insights while fresh
- Update timestamps
- Preserve existing content, don't overwrite unnecessarily
